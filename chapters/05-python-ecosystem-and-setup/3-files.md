# Python-Skripte: Dateien strukturieren

Im vorherigen Abschnitt haben wir eine `.py`-Datei erzeugt und ausgeführt – also ein Programm gestartet. In der Praxis (und auch im Kurs) wächst Code schnell über „eine Datei mit ein paar Zeilen“ hinaus: Sie schreiben mehrere Programme/Varianten, möchten Teile wiederverwenden und wollen dabei den Überblick behalten.

In diesem Teil geht es deshalb nicht um „mehr Syntax“, sondern um eine praktische Grundidee: **Skripte haben einen Startpunkt (Entrypoint), einen Ablauf (Workflow) und Bausteine (Funktionen/Module)**.
